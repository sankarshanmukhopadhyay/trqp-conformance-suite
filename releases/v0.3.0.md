# TRQP Conformance Suite v0.3.0

This release delivers the Commit 3 and Commit 4 hardening increments for the Conformance Suite.

## Highlights

- Add `--dry-run` flag: validates inputs and lists applicable tests without making HTTP requests.
- Add `--list-tests` flag: prints tests applicable to the selected profile and exits.
- Add `identifiers:` block support in `sut.yaml` so real SUTs can provide their own `authority_id`, `entity_id`, `subject_authority_id`, and `action` values without modifying core test definitions.
- Fix JSONPath wildcard token indexing bug that caused incorrect path resolution when the wildcard token was encountered more than once.
- Refactor `cts/run.py` by extracting `resolve_identifiers`, `apply_identifier_overrides`, and `list_tests` as named functions.
- Add `QUICKSTART.md` for fast onboarding.
- Expand `SECURITY.md` with threat model references and reporting scope clarification.
