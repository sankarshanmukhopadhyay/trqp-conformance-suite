# TRQP Conformance Suite v1.6.0 — Reproducible Conformance Evidence

## Summary

This release makes the Conformance Suite a governed evidence producer for the coordinated TRQP End-to-End Assurance Execution and Evidence Chain.

## Coordinated release tuple

| Component | Version | Role |
|---|---:|---|
| TRQP Assurance Hub | v1.9.0 | Evidence composition and assurance decision |
| TRQP Conformance Suite | v1.6.0 | Protocol conformance evidence |
| TRQP-TSPP | v0.14.0 | Security and privacy posture evidence |

## Added

- Machine-readable `PROJECT-STATUS.yaml` and schema validation.
- Fixture-backed conformance report generation bound to explicit run and target identity.
- Requirement coverage, negative-test coverage, and SHA-256 evidence indexing.
- `make assurance-check` and `make evidence` entry points.
- Cross-stack execution guidance.

## Known result

The checked-in baseline fixture currently records seven passing and four failing assertions. This is retained intentionally: the Hub must propagate genuine producer failures rather than manufacture a passing assurance conclusion.

## Compatibility

This release is coordinated with Hub v1.9.0 and TSPP v0.14.0 under release set `ots-2026-07`.

## Validation

```bash
make validate
make assurance-check
```

## Authority and limitations

CTS owns executable conformance tests, verdict semantics, and conformance evidence. It does not own the TRQP protocol specification or external certification decisions.

## Upgrade guidance

Run the new assurance target, inspect `artifacts/validation/cts-report.json`, and remediate failing fixture/profile assertions before representing the target as conformant.
