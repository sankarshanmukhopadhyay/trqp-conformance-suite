# TRQP Conformance Suite v0.8.0

This minor release fixes schema contract defects that caused the CTS reference SUT and
TSPP harness to be non-interoperable, adds freshness requirements to the Enterprise
profile, and introduces the Ayra Trust Network baseline profile.

## Highlights

- **`recognised` → `recognized`**: the recognition response schema and reference service
  now use the ToIP-canonical US spelling, matching TSPP and the Ayra TRQP Profile API.
- **Freshness envelope**: `authz_response` and `recog_response` schemas now accept and
  validate a `meta` object with `time_evaluated` and `expires_at`. The reference SUT
  emits these fields, making it passable against both CTS and the TSPP AL1 harness
  without modification.
- **Error schema hardening**: the `error` field is now constrained to the TSPP-aligned
  enum, providing a stable machine-readable error surface.
- **Ayra baseline profile**: new `profiles/ayra_baseline.yaml` for operators targeting
  the Ayra Trust Network. Extends Enterprise with freshness and recognition gates.
- **Broken link fix**: `docs/assurance-levels.canonical.md` back-link now resolves correctly.
- **Version pin fix**: `docs/hub-crosswalk.md` now matches `al-contract.json` (Hub v0.9.0).

## Breaking changes

- `schemas/core/recog_response.schema.json`: `statement.recognised` → `statement.recognized`
  and `statement.recognised_since` → `statement.recognized_since`. Servers emitting the
  old UK-spelling keys will fail TC-RECOG-001 schema validation.
- `schemas/core/error.schema.json`: `error` field now has an enum. Error responses using
  codes outside the declared list will fail schema validation. The enum includes all
  codes emitted by the reference SUT and TSPP.

## Migration guide

**Implementers upgrading from v0.7.x:**

1. Rename `statement.recognised` → `statement.recognized` in recognition responses.
2. Rename `statement.recognised_since` → `statement.recognized_since`.
3. Add `meta: { time_evaluated: "<RFC3339>", expires_at: "<RFC3339>" }` to all responses
   (required for Enterprise profile; SHOULD for Baseline).
4. Ensure error responses use one of the declared `error` enum values.
