# v1.1.0

Deterministic Replay and Fixture-Pinned Runs — Release Set A.

## Included

- **`--generated-at` flag**: pins all `generated_at` timestamps in `run.json`, `cts-report.json`, `manifest.json`, and `checksums.json` to a caller-supplied value, making evidence artifacts bit-reproducible across identical inputs.
- **`--fixture-set <file>` flag**: runs the suite against canned JSON responses instead of a live SUT. Fixture set SHA-256 and `fixture_set_id` are embedded in `run.json` for complete input provenance. Missing fixture entries produce `SKIP` with a clear reason rather than silently erroring.
- **`--replay <run-dir>` flag**: re-evaluates all assertion logic against captured `cases/*.json` files without making any HTTP requests. Emits `replay-report.json` with a `replay_of_run_id` back-pointer and a `verdict_diffs` list showing which test cases changed result between the original and replay run.
- **`fixtures/baseline.fixture-set.json`**: canonical fixture set for the `baseline` profile, covering all 11 non-high-assurance test cases with responses that are valid against their expected schemas. Suitable for CI snapshot pinning and third-party audit replay.
- **AL contract hash reconciliation**: fixed `docs/assurance-levels.canonical.md` back-link and updated `al-contract.json` to pin the correct canonical doc SHA-256 (`61c599c5...`), aligning with TRQP Assurance Hub v1.2.0 and TRQP-TSPP v0.8.0.
- **CI update**: fixture-pinned baseline run and live-run replay are now both executed on every push to `main`, with artifacts uploaded separately for audit access.
